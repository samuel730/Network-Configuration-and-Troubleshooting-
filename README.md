# Network-Configuration-and-Troubleshooting-
## Introduction

In modern DevOps environments, reliable network configuration and troubleshooting are essential for maintaining system availability, communication between services, and access to external resources. Linux systems form the backbone of most cloud and infrastructure platforms, making it important to understand how network interfaces, routing tables, and DNS resolution work at a low level.

This project focuses on hands-on network configuration and troubleshooting using a Linux (Ubuntu 18.04) Vagrant virtual machine. It demonstrates how to identify network interfaces, diagnose connectivity issues, correct misconfigured routing, and verify internet and DNS functionality using standard Linux networking tools such as `ip`, `ping`, and `dig`.

By completing this lab, you gain practical experience in resolving real-world network issues that commonly occur in DevOps and system administration environments, strengthening your ability to maintain and debug infrastructure systems effectively.
